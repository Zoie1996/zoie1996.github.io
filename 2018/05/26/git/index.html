Git的基本常用命令(入门级)

基本命令

  • git –version 查看git版本

  • git config –global user.name ‘xxx’ 配置用户名

  • git config –global user.email 'xxx@xxx.com‘ 配置用户邮箱

  • git init 初始化

  • git add [file] 把文件添加到版本控制

  • get add . 把当前路径下所有文件纳入暂存区

  • git commit -m ‘说明’ 将文件放入仓库

  • git status 查看放在暂存区的文件状态

  • git checkout – 撤回所有暂存区文件

  • git checkout – [file] 撤回暂存区文件

  • git log 查看历史版本

  • git log –pretty=oneline 单行显示历史版本

  • get reflog 查看版本包括’’未来’’版本

  • git reset –hard [版本代号前6位] 返回任何一个版本

git版本控制流程

本地建仓库 → 托管到远端

  • 创建一个空项目 → 获取url
  • mkdir [dir] 创建一个新的文件夹
  • cd [dir] 切换到文件夹下
  • git init 初始化
  • 添加文件到新创建的文件夹下
  • git add . 将文件放入暂存区
  • get status 查看放在暂存区的文件状态
  • git commit -m ‘说明’ 将文件放到仓库
  • git reflog git remote add origin \<url> 连接接到远端仓库
  • git push [-u] origin master 将本地仓库与远端仓库origin别名对应 第一次提交代码需要输入 -u
  • git pull 获取服务器更新的代码

远端服务器项目已存在

  • git clone [url] 下载远端服务器上的文件
  • cd [dir] 切换到文件夹下
  • git branch [分支file] 创建分支名
  • git branch 查看分支
  • git checkout [分支file] 切换到分支
  • 更新 / 添加 / 删除内容
  • git add . 将文件放到暂存区
  • git commit -m ‘说明’ 将文件提交仓库
  • git push origin [分支file] 将文件推送到远端服务器分支
  • get checkout master 切回到master
  • git merge [分支file] 将分支合并到master
  • git push origin master 将本地项目推送到服务器
